# Atlas Version History

This file is the human-readable companion to the remote `atlas_versions.parquet`
registry hosted in the `cellNexus-metadata` Nectar object store. Each entry here
mirrors a row in that registry and is added alongside every call to
`cellNexus:::register_atlas_version()`. The maintainer follows the atlas version
in this README to append new registry.

## Semantic versioning

The `atlas_id` column in `get_metadata()` is generated by the form
`<collection>_<census-year>/<X.Y.Z>`, e.g. `cellxgene_2024/0.1.0`.

| Version bump | When to use |
|---|---|
| **patch** `0.1.x` | Bug fix within the same data (e.g. negative count values corrected) |
| **minor** `0.x.0` | File IDs regenerated, new datasets added, or non-breaking schema additions |
| **major** `x.0.0` | Breaking format change (e.g. H5AD → TileDB) or a new Census year |

## Adding a new entry
  
When a new data version is released, run the following in R:
  
```r
cellNexus:::register_atlas_version(
  atlas_id       = "cellxgene_YEAR/0.2.0",
  census_version = "DD-MM-YY",
  container      = "cellNexus-metadata",
  change_type    = "minor",
  description    = "Describe what changed.",
  credential_id     = Sys.getenv("NECTAR_CREDENTIAL_ID"),
  credential_secret = Sys.getenv("NECTAR_CREDENTIAL_SECRET")
)
```
This appends a new entry to the file stored on the Nectar object store and refreshes the corresponding data in the cloud.
